# Пагинация на странице проектов

## Описание

Добавлена пагинация на страницу `/projects` для улучшения производительности и удобства просмотра большого количества проектов.

## Основные изменения

### 1. Параметры пагинации

- **Количество проектов на странице**: 7
- **Текущая страница**: отслеживается в состоянии `currentPage`
- **Автоматический сброс**: при изменении фильтров страница сбрасывается на первую

### 2. Новые функции

#### `getPaginatedProjects(projects)`
Возвращает проекты для текущей страницы:
- Применяет фильтрацию и сортировку
- Вычисляет диапазон элементов для текущей страницы
- Возвращает срез массива проектов

#### `getTotalPages(projects)`
Вычисляет общее количество страниц на основе отфильтрованных проектов.

#### `goToPage(page)`
Переход на указанную страницу:
- Обновляет `currentPage`
- Плавно прокручивает страницу вверх

#### `nextPage(projects)`
Переход на следующую страницу (если доступна).

#### `prevPage()`
Переход на предыдущую страницу (если доступна).

### 3. UI компоненты пагинации

#### Информация о результатах
```
Показано 1–7 из 25 (отфильтровано из 100)
```

#### Кнопки навигации
- **Предыдущая страница**: стрелка влево (отключена на первой странице)
- **Номера страниц**: 
  - Всегда показываются первая и последняя страницы
  - Показываются страницы рядом с текущей (±1)
  - Многоточие между группами страниц
  - Текущая страница выделена цветом
- **Следующая страница**: стрелка вправо (отключена на последней странице)

### 4. Автоматический сброс страницы

При изменении любого из фильтров страница автоматически сбрасывается на первую:
- Поиск (`searchTerm`)
- Статус (`statusFilter`)
- Период (`dateFilter`, `dateFrom`, `dateTo`)
- Сортировка (`sortBy`, `sortOrder`)

Реализовано через `$effect()` в Svelte 5.

## Примеры использования

### Навигация по страницам

1. **Переход на конкретную страницу**:
   - Нажмите на номер страницы в пагинаторе

2. **Последовательная навигация**:
   - Используйте кнопки со стрелками для перехода вперед/назад

3. **Быстрый переход**:
   - Первая и последняя страницы всегда доступны для клика

### Поведение при фильтрации

1. Примените фильтр (например, выберите статус)
2. Страница автоматически сбросится на первую
3. Пагинация пересчитается для отфильтрованных результатов

## Технические детали

### Производительность

- Фильтрация и сортировка выполняются один раз
- Пагинация применяется к уже отсортированному массиву
- Используется `slice()` для эффективного получения среза данных

### Адаптивность

- На мобильных устройствах пагинация адаптируется под размер экрана
- Кнопки имеют минимальный размер для удобного нажатия
- Информация о результатах переносится на новую строку на маленьких экранах

### Доступность

- Все кнопки имеют `aria-label` для скринридеров
- Текущая страница помечена `aria-current="page"`
- Отключенные кнопки имеют `disabled` атрибут

## Стилизация

Пагинация использует существующую цветовую схему проекта:
- **Активная страница**: cyan-600 фон, cyan-500 граница
- **Неактивные страницы**: gray-700 фон, gray-600 граница
- **Hover эффект**: gray-600 фон
- **Отключенные кнопки**: opacity-50
